アイデアスケッチ


今回は①の競馬ゲームの制作にチャレンジします！
※変更の可能性はゼロではない。

概要

「あの熱狂を、手元に。」をコンセプトに、競馬愛を詰め込んだ作品にします。
※概要も随時変更を加えていく予定です。

プロセスメモ

■=考えないといけないことや疑問など　●=具体的にやること　→=やったこと

■ IoTをどのように活用するか考える。
→ システムの開始にQRコードを使う（今のところ）
● dflinkを使ったQRコードのシステムを動作に組み込む。
→ Pygameがデータの変化をうまく読み取ってくれない...
● QRコードの読み取りでPygameの動作を開始できるようにがんばる。

■ ArduinoとPythonをうまく同時併用できるのか？
● Lチカで代用しながら↑のようなプログラムが動くか確かめる。
→ Pythonの方でシリアル入力＆特定のワードをキャッチすることで連動した！
● モーターの動作が上手くいき次第あらためて同時に動かしてみる。
→ モーターは動作したが、その起爆剤であるPygameが全然うごかない。なんとかしたい。

● 着順掲示板をPythonで再現する。
→ Pygameを使って、作成した画像上に文字を出すことに成功！

● プログラムをいじって動きを作る。
→ テキストごとに表示タイミングを変更して順番に表示されるようにした。
● なんか寂しいので効果音を導入する。
→ Pygame.mixerを使って音を鳴らすことに成功！ファンファーレや歓声があっていいかんじ

● 購入する部品をリストアップする。
→ DCモーター、DCモータードライバ、電池ボックスを買う！
→ 電池ボックスは単三１本のものから２本のものに交換！
■ 安全に動作する回路を組む。
→ 下の図の回路を組んだ！...ものの動作しなかったので組み換え部品交換などを検討。
● モーターが動作するよう回路を改善する。
→ 回路を組みなおしても動作しないので、モータードライバをTC78H653FTGからTB6612に変更してみる。
→ 無事にモーターが動いた！
● モーターの速さを徐々に変えるプログラムを書く。
→ PWMAの値を変化させることでスピードを変化。
● 電池が異常に発熱してしまったので、安全に動作・停止するようコードを工夫する。（原因はスタンバイモードの設定ミス？）
→ 念のためにモーター動作中以外はスタンバイモードをオフにする。

■ 作品のガワと内部機構の考えをまとめる。
● どのような構造で競走馬を動かすか考える
→ 歯車にカムを付ける形で上下させる！
● 作品本体のサイズ感を決める。（他の作品がデカそうなのでちょっと大きめにしたい...）
→ 画像のようなサイズ感で制作していく。ほどよい。

● Fusionで競走馬を動かす機構を設計してテストする。
→ 歯車部分のプロトタイプが完成！動作に問題ないので量産したい。

● 全パーツが完成したので組み立てる。
→ 本体組み立て完了！干渉もせずちょうどいいサイズ感。
